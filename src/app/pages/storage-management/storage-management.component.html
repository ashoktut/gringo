info About Storage Migration

Why IndexedDB?

  • Much larger storage capacity (hundreds of MBs vs 5-10MB)
  • Better performance for large datasets
  • Asynchronous operations don't block the UI
  • Supports complex data types and binary data
  • Transactional operations prevent data corruption

Migration Process

  • Data is automatically migrated when services start
  • Original localStorage data is preserved during migration
  • You can manually clear localStorage after successful migration
  • All existing functionality continues to work seamlessly

Browser Support

  • Supported in all modern browsers (Chrome, Firefox, Safari, Edge)
  • Graceful fallback to localStorage if IndexedDB fails
  • Data persists between browser sessions
  • Not affected by private/incognito browsing limitations
